## 问题

字段提取任务里，如何评估标注数据质量？标注成什么样算合格？数据怎样混合？

## 答案

> 🤖 AI 答案 · 模型：Claude-Opus-4.8 · 日期 2026-06-23

字段提取（如从合同/票据/简历抽 `金额`、`日期`、`公司名`）的标注质量，要分三层把关：**一致性、准确性、合格门槛**，再谈数据混合。

**1. 一致性（标注者之间稳不稳）**

让多人标同一批样本，量化一致程度：

- **分类型字段**（是否命中、字段类型）用 **Cohen's / Fleiss' Kappa**，扣除随机一致后看真实一致性，一般 **κ≥0.8 优秀、0.6~0.8 可接受、<0.6 须返工**。
- **抽取片段（span）** 用 **F1 / 精确匹配率**：边界、归一化（`2026/6/23` vs `2026-06-23`）是否对齐。
- 一致性低往往不是标注员的错，而是**标注规范（guideline）有歧义**——要回头补规则、补反例。

**2. 准确性（标得对不对）**

一致 ≠ 正确，还要对**金标准（gold set）**：

```
质检三件套
 ├─ 金标对比：专家标的小批 gold，算 P/R/F1，看绝对准
 ├─ 随机抽检：按批次抽 5~10%，错误率 = 错字段数 / 抽检字段数
 └─ 程序校验：日期/金额/枚举正则与业务规则兜底，挑硬伤
```

逐**字段**算指标，别只看整体——`金额`常因单位/小数出错，`公司名`常因简称/全称不一。

**3. 合格标准（什么样算过）**

按字段重要度分级定阈值，例如：核心字段 **F1≥0.95**、一般字段 ≥0.90；批次抽检**字段级错误率＜2%**、且无系统性错误（同类错反复出现）。不达标的批次**整批退回重标**，而不是只改抽到的那几条。

**4. 数据怎么混合**

训练集要可控地配比，避免模型偏科：

| 维度 | 混合原则 |
| --- | --- |
| 难易 | 简单/中等/困难（模糊、缺失、嵌套）按约 5:3:2，保证有硬样本 |
| 来源 | 多模板/多版式/多渠道按线上真实分布配比，防过拟合单一模板 |
| 正负 | **保留负例与空值**：字段不存在、易混淆字段，正负适度均衡，教会模型「该不抽就不抽」 |

```
最终训练集 = 难易分层 × 来源覆盖 × 正负均衡
            └─ 贴近线上分布，但对长尾/难例适度上采样
```

**一句话总结：** 一致性（Kappa）锁规范、金标+抽检锁准确、分字段定合格阈值；数据按难易/来源/正负三轴混合并贴近线上分布，长尾难例适度上采样。

## 延伸 / 追问

**追问：标注成本有限，怎么把质检和标注资源花在刀刃上？**

抓两点。其一，**主动学习 + 难例优先**：先用小模型/规则跑一遍，把模型低置信、字段冲突、校验报错的样本优先送人工标与质检，简单样本可机标+轻抽检，省下的人力压在长尾。其二，**分级质检而非全量复核**：核心字段、新模板、新标注员的产出 100% 或高比例抽检，成熟稳定的流水线降到 5%，并用金标"埋点"持续监控标注员漂移。再配标注规范 + 典型错例库做培训，让一致性从源头提上来——返工一次的成本远高于一开始把 guideline 写清楚。
